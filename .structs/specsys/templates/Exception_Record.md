<a id="owner"></a>
## Owner

Named accountable owner. Role-only ownership is not sufficient for an active exception.

<a id="affected-control"></a>
## Affected Control

Policy section, control ID, template ID, CI gate, security control, or release criterion affected.

<a id="reason"></a>
## Reason

Specific justification. Convenience, schedule pressure, or incomplete evidence alone are not sufficient.

<a id="scope"></a>
## Scope

Repository, service, environment, version, asset, user group, data class, and time period. Scope MUST be bounded.

<a id="compensating-control"></a>
## Compensating Control

Temporary control that reduces risk until the exception is remediated.

<a id="expiry-date"></a>
## Expiry Date

YYYY-MM-DD. Open-ended exceptions are prohibited.

<a id="required-remediation"></a>
## Required Remediation

Required action, owner, target date, validation command, and evidence link.
